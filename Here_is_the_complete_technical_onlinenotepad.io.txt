8. MODEL DESIGN

The suggested Intelligent WAF is based on deep learning models that identify normal and malicious HTTP requests. When the requests in the HTTP are sequential text patterns, the models that can learn sequential relationships can be used for this task. In this project, three models are designed and evaluated: LSTM baseline, Bi-LSTM with attention, and Transformer. The proposed model is Bi-LSTM with attention model due to its good overall performance.



B. LSTM Explanation Model

As the first sequence-learning model, LSTM is used to make a comparison. The LSTM network can be used for long-term memory to process sequential data, and is appropriate for processing the structure of HTTP request sequences on characters. This model uses an embedding layer, followed by an LSTM layer for each tokenized request. The LSTM representation is then added to the numerical feature vector and fed to fully connected layers for binary classification.



The LSTM model serves as a baseline to assess the validity of the potential of bidirectional sequence processing and attention for detection.



B. Proposed Bi-LSTM with Attention Model

The proposed model is a Bidirectional Long Short-Term Memory (BLSTM) network with attention. A Bi-LSTM is the same as a standard LSTM but takes into account the input sequence from both forward and backward direction. This enables the model to create context of previous and subsequent characters of the HTTP request.



In a malicious payload, attack signs could be at the start, center, or end of the request. These dependencies can be better modeled using a bidirectional model as opposed to a single-direction LSTM.



The proposed Bi-LSTM model has the following layers:



Embedding Layer



The character level sequence of tokens is mapped to dense vectors. The embedding vector for each token is a 64-dimensional vector. The padding index is used to ignore padding tokens.



Bidirectional LSTM Layer



A two-layer Bi-LSTM is used to pass the embedded sequence through. There are 128 hidden units for each LSTM direction. Model is bi-directional, which means that the output representation will include forward and backward information.



Attention Layer



The attention mechanism assigns importance weights to different positions in the sequence. This will help the model to be focused on suspicious characters or substrings like SQL keywords, script tags, encoded characters, traversal patterns, and the command separators.



Feature Fusion Layer



The concatenated Bi-LSTM output and eight engineered numerical features are fed into the attended LSTM. This enables the model to learn sequence features and manually extract structural features.



Fully Connected Classifier



Input is a combined feature vector, which is passed through fully connected layers equipped with ReLU activations, batch normalization and dropouts. The last layer applies sigmoid activation function to output an attack probability that is in the range of 0-1.



If the predicted probability is greater than or equal to 0.5, the request is considered an attack. If not, it is considered to be normal.



C. Transformer Model

In addition, a Transformer model is implemented to make a comparison. Transformer architectures employ self-attention to learn relationships among positions in a sequence. The following components are used in this project: character embeddings, positional encoding, multi-head attention, encoder layers, numerical feature fusion, and Transformer model.



Although the Transformer model performs strongly, it does not outperform the Bi-LSTM model in this project. This is possibly due to the fact that the CSIC 2010 dataset is of moderate size; the Bi-LSTM model is more suitable to learn sequential request patterns from such a dataset.



D. Hybrid Detection Strategy

The deployed WAF is not just based on the neural model. It adopts a mixed approach:



Rule-Based Layer



Known attack signatures are checked first using regular expressions. These include SQL injection, XSS, path traversal and command injection patterns. If a match is identified, the request is rejected straight away.



Deep Learning Layer



When there is no rule matching, the request goes to the Bi-LSTM attention model. The end classification is done by model using feature of characters level and numerical characteristics of the request.



Such a hybrid design enhances its usability in real-world applications as known attacks can be identified swiftly, and the Bi-LSTM allows for adaptability in scenarios involving modified or new patterns.
