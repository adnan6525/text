 XI. CONCLUSION
This project presented an Intelligent Web Application Firewall using deep learning for detecting malicious HTTP requests and zero-day-like web attack patterns. Traditional WAFs such as ModSecurity depend mainly on predefined rules and signatures, which makes them effective against known attacks but less reliable against modified, obfuscated, or previously unseen payloads. To address this limitation, the proposed system combines rule-based filtering with a Bi-LSTM deep learning model using an attention mechanism.
The CSIC 2010 HTTP dataset was used for training and evaluation. HTTP requests were represented using character-level tokenization and eight engineered numerical features. Three deep learning models were developed and compared: LSTM baseline, Transformer, and Bi-LSTM with attention. Among these models, the proposed Bi-LSTM achieved the best performance with 96.02% accuracy, 99.53% precision, 90.72% recall, 94.92% F1-score, 99.50% ROC-AUC, and a 0.30% false positive rate.
The results show that the Bi-LSTM model outperforms ModSecurity CRS v3, LSTM, and Transformer models in overall detection performance. The reduction in false positive rate is especially important for WAF deployment because it reduces unnecessary blocking of legitimate users. The Flask-based prototype further demonstrates that the trained model can be integrated into a real-time inspection system capable of producing block/allow decisions, confidence scores, and request logs.
Overall, the project confirms that deep learning can improve web application firewall performance by learning attack patterns from HTTP request data. The hybrid approach provides the benefits of fast rule-based detection for known attacks and adaptive deep learning-based classification for modified or zero-day-like attack patterns.





XII. FUTURE SCOPE
The proposed Intelligent Web Application Firewall demonstrates strong performance on the CSIC 2010 HTTP dataset and provides a working Flask-based prototype. However, several improvements can be made in future work to increase its real-world applicability, scalability, and robustness.
1. Training on Real-World Traffic The current model is trained mainly on benchmark HTTP request data. Future work can include real-world web server logs and production traffic to improve generalization against modern attack patterns.
2. Support for More Attack Categories The system can be extended to detect additional attack types such as server-side request forgery, remote code execution, file upload attacks, authentication bypass, API abuse, and bot-based attacks.
3. Advanced Tokenization Techniques Future versions can explore word-level, subword-level, or byte-pair encoding tokenization to capture semantic patterns more effectively than simple character-level tokenization.
4. Explainable AI for WAF Decisions Explainability methods can be added to show which parts of a request influenced the model’s decision. This would help security analysts understand why a request was blocked or allowed.
5. Online and Incremental Learning The model can be improved with online learning so that it can adapt to new attack patterns over time without complete retraining.
6. Adversarial Robustness Future work can test the model against adversarial payloads designed to bypass deep learning-based detection. Defensive methods can then be added to improve robustness.
7. Cloud and Enterprise Deployment The prototype can be deployed as a reverse proxy or middleware in front of real web applications. It can also be integrated with cloud platforms, load balancers, and containerized environments.
8. SIEM and Alert Integration The WAF can be connected with Security Information and Event Management systems to provide centralized logging, alerting, and incident response support.
9. Latency and Performance Optimization Model compression, pruning, quantization, or ONNX conversion can be used to reduce inference time and make the WAF suitable for high-traffic production environments.
10. Continuous Rule and Model Updating
A future system can combine automatic rule updates with periodic model retraining so that both signature-based and learning-based components remain effective against evolving threats.
