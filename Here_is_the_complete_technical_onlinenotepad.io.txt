 XI. CONCLUSION
This project introduces an Intelligent Web Application Firewall based on deep learning to detect malicious HTTP requests and web attack patterns similar to zero-day attack. The traditional WAFs like ModSecurity rely primarily on rules and signatures, and are effective against known attacks, but have difficulties in detecting modified, obfuscated and unknown payloads. To overcome this drawback, the proposed system is based on a combination of rule-based filtering with a Bi-LSTM deep learning model with an attention mechanism.
Training and evaluation were done using the CSIC 2010 HTTP dataset. Character level tokenisation and 8 engineered numerical features were used to represent HTTP requests. A total of three deep learning models were created and evaluated: LSTM baseline, Transformer and Bi-LSTM with attention. The proposed Bi-LSTM model gave the highest accuracy performance of 96.02%, followed by precision of 99.53%, recall of 90.72%, F1 Score of 94.92%, ROC-AUC of 99.50%, and, Finally FP Rate of 0.30% among these models.
The results demonstrate that the Bi-LSTM model achieves the best overall performance in terms of detection performance, compared to ModSecurity CRS v3, LSTM and Transformer models. An important consideration for WAF deployment is the reduction in false positive rate, which will minimize the blocking of legitimate users. The prototype system further demonstrates that the trained model can be embedded in a real-time inspection system that can give block/allow decisions, confidence scores and request logs.
Overall, the project shows that deep learning can improve the performance of a web application firewall by learning the patterns of attacks on the HTTP data of the application. Hybrid approach combines advantages of rapid attack detection by the rule-based classifier for known attacks with the advantages of the modified or zero-day-like attacks classification by the deep learning classifier.





XII. FUTURE SCOPE
The idea of the Intelligent Web Application Firewall shows good performance on the CSIC 2010 HTTP dataset and offers a working prototype of an Intelligent Web Application Firewall using Flask. Several enhancements and extensions are suggested for future research so as to enhance its applicability, scalability and robustness in real time.
The current model is trained primarily on benchmark data in the form of an HTTP request. Real-world web server logs and production traffic could be used in the future to further generalize against current attack patterns.
The system can be extended to support other attack types like server-side request forgery, remote code execution, file upload attacks, authentication bypass, API abuse, and bot-based attacks.
3. Advanced Tokenization Techniques Word-level, subword-level and byte-pair encoding tokenizations could be used in future versions to better capture semantic patterns than character-level tokenization.
Explainable AI for WAF Decisions: Methods of explainability can be integrated to reveal which components of a request had an impact on the decision. This would make it easier for security analysts to know the reason for the security request being denied or accepted.
Online and incremental learning: The model can be continuously improved through online learning, thus it can learn new attack patterns over time without the need for retraining the whole model.
An adversarial payload is a future test case for the model that can evade detection through deep learning. Robustness can then be increased through the use of defensive methods.
The prototype can run as a reverse proxy or middleware in front of the real web applications. It can also be embedded in cloud environments, load balancers and containerized environments.
8. SIEM and Alert Integration: The WAF can integrate with SIEM systems to enable centralized logging, alerting and incident response functions.
Efficiency can be achieved by model compression, pruning, quantization, or ONNX conversion to lower inference time and scale the WAF for high traffic production environments.

A future system could integrate with automatic updates to a signature-based component, and periodically retrain the learning-based component to ensure that both are resilient against threats.
