D. Prediction Pipeline
The prediction pipeline is implemented in firewall/predict.py. It performs the following operations:
1. Receives URL, request content, and HTTP method.
2. Applies rule-based signature checking.
3. Extracts numerical features from the request.
4. Converts URL and content into a character-level token sequence.
5. Passes the sequence and numerical features to the trained Bi-LSTM model.
6. Generates label, confidence score, attack probability, extracted features, and detection method.
If the rule-based layer detects a known attack signature, the system immediately returns an attack label with high confidence. If no rule is matched, the Bi-LSTM model performs the final classification.

  10. RESULTS AND DISCUSSION
The proposed Intelligent WAF is evaluated using standard binary classification metrics, including accuracy, precision, recall, F1-score, ROC-AUC, and false positive rate. The performance of the proposed Bi-LSTM model is compared with the LSTM baseline, Transformer model, and ModSecurity CRS v3.
A. Evaluation Metrics
The following metrics are used for performance evaluation:
* Accuracy: Measures the overall percentage of correctly classified requests.
* Precision: Measures how many requests classified as attacks are actually attacks.
* Recall: Measures how many actual attacks are correctly detected.
* F1-score: Harmonic mean of precision and recall.
* ROC-AUC: Measures the model’s ability to distinguish between normal and attack traffic across thresholds.
* False Positive Rate (FPR): Measures the percentage of normal requests incorrectly classified as attacks.
For a WAF system, false positive rate is especially important because incorrectly blocking legitimate users can reduce usability and affect business operations.

 C. Discussion of Results
The proposed Bi-LSTM model achieves 96.02% accuracy, which is higher than the LSTM baseline, Transformer model, and ModSecurity CRS v3. The Bi-LSTM also achieves the highest precision of 99.53%, showing that very few normal requests are incorrectly classified as attacks. The Bi-LSTM model achieves an F1-score of 94.92%, indicating a strong balance between precision and recall. Its ROC-AUC value of 99.50% shows that the model can effectively distinguish between normal and malicious HTTP requests across different classification thresholds.
The LSTM baseline performs significantly lower than the proposed Bi-LSTM. This shows that single-direction sequence processing is not sufficient to capture all request patterns effectively. The Transformer model performs better than the LSTM baseline but still remains below the Bi-LSTM. This may be because the CSIC 2010 dataset is moderate in size, and the Bi-LSTM model is more effective for learning sequential character-level patterns in this dataset.
